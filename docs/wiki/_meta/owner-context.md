# Owner Context Boundary

## Purpose

Open Scaffold is an open-source project. Public repo artifacts should describe the project in owner-neutral language and should not depend on private owner biography or private operational systems.

## Boundary

This project wiki may mention that private owner-level systems exist, but it should not copy their raw content, personal context, private execution state, or private strategy.

When owner perspective matters, use neutral wording such as "the owner" or "the project owner".

## Relationship to This Wiki

This wiki stores public-safe compiled project knowledge: ontology, durable principles, comparisons, and query answers. Live project execution stays in `.osc/`, GitHub, evidence/release notes, and PR review surfaces.
